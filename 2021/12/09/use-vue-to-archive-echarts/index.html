Vue组件化逻辑实现Echarts

2021-12-09
2021年-Vue

使用背景

使用Vue框架进行前端页面编辑，利用Vue的模块化前端思维，将主页分拆成多个组件进行单独的逻辑实现，最终在主页进行组合。由于项目需要从后台读取数据库数据进行前端呈现，因此Echarts需要接受API接口数据，而不是静态展示数据。

以Vue-element-admin框架为例子，进行逻辑实现

实现逻辑

目录树

实现逻辑

  1. 项目的Vue文件以目录作为分类，外层vue文件是内层组件的父组件，数据从外传递至内层子组件；
  2. 第一个index.vue文件通过角色控制显示内容；
  3. admin文件夹内的index.vue控制admin角色显示的内容，以及以标签模板的方式向子组件传递的数据；
  4. LineChart.vue 变量chatdata接收父组件（admin/index.vue）以lineChartData变量名传递的数据；

组件实现详解

LineChart组件

实现代码

admin/components/LineChart.vue
<template>
  <div :class="className" :style="{height:height,width:width}" />
</template>
<script>
import echarts from 'echarts'
require('echarts/theme/macarons') // echarts theme
import resize from './mixins/resize'
//import {getdate} from './mixins/getdate'
export default {
  mixins: [resize],
  props: {
    className: {
      type: String,
      default: 'chart'
    },
    width: {
      type: String,
      default: '100%'
    },
    height: {
      type: String,
      default: '350px'
    },
    autoResize: {
      type: Boolean,
      default: true
    },
    chartData: {
      type: Object,
      required: true
    },
    date:{
      type:Array,
      required:true
    }
  },
  data() {
    return {
      chart: null
    }
  },
  watch: {
    chartData: {
      deep: true,
      handler(val) {
        this.setOptions(val)
      }
    }
  },
  mounted() {
    this.$nextTick(() => {
      this.initChart()
    })
  },
  beforeDestroy() {
    if (!this.chart) {
      return
    }
    this.chart.dispose()
    this.chart = null
  },
  methods: {
    initChart() {
      this.chart = echarts.init(this.$el, 'macarons')
      this.setOptions(this.chartData)
    },
    setOptions({ Data } = {}) {
      this.chart.setOption({
        xAxis: {
          data: this.date,
          boundaryGap: false,
          axisTick: {
            show: false
          }
        },
        yAxis: {
          axisTick: {
            show: false
          }
        },
        series: [
        {
          name: 'data',
          smooth: true,
          type: 'line',
          data: Data,
          animationDuration: 1500,
          animationEasing: 'quadraticOut'
        }]
      })
    }
  }
}
</script>

简单实现逻辑

  1. LineChart组件接收父级组件（admin/index.vue）通过绑定模板传递的数据chart-data；
  2. 子级组件linecharts通过prop获取父级组件的数据，
  3. 通过mount方法在数据更新后初始化Echarts 折线图；
  4. 折线图通过调用vue提供的watch方法，监控PanelGroup的点击事件，更新折线图的数据Data

        </div>

import CountTo from 'vue-count-to'

简单实现逻辑

  1. 通过接收父级组件模板绑定的数据，子级组件在模板中实现动态变化
  2. 通过$emit 和模板中onclick的事件监控，实现数据向子级组件的传递

BarChart

实现代码

<template>
  <div :class="className" :style="{height:height,width:width}" />  
</template>

<script>
import echarts from 'echarts'
require('echarts/theme/macarons') // echarts theme
import resize from './mixins/resize'
import { getBardata } from '@/api/barchats'

const animationDuration = 6000
const option = {  //EChats实例的配置参数
        tooltip: {
          trigger: 'axis',
          axisPointer: { // 坐标轴指示器，坐标轴触发有效
            type: 'shadow' // 默认为直线，可选为：'line' | 'shadow'
          }
        },
        grid: {
          top: 10,
          left: '2%',
          right: '2%',
          bottom: '3%',
          containLabel: true
        },
        xAxis: {
          type: 'category',
          data: [],
          axisTick: {
            alignWithLabel: true
          }
        },
        yAxis: [{
          type: 'value',
          axisTick: {
            show: false
          }
        }],
        series: [{
          name: '分数',
          type: 'bar',
          stack: 'vistors',
          barWidth: '60%',
          data: [],
          animationDuration
        }]
      }

export default {

  mixins: [resize],
  props: {
    className: {
      type: String,
      default: 'chart'
    },
    width: {
      type: String,
      default: '100%'
    },
    height: {
      type: String,
      default: '300px'
    },
    
  },
  data() {
    return {
      chart: null,
    }
  },
  mounted() {
    this.$nextTick(() => {
      this.initChart()
    })
    //this.initChart()
  },
  beforeDestroy() {
    if (!this.chart) {
      return
    }
    this.chart.dispose()
    this.chart = null
  },
  methods: {
    initChart() {
      // 准备好dom，初始化echarts实例
      this.chart = echarts.init(this.$el, 'macarons')
      this.chart.setOption(option)
      // 通过API组件获取数据，并将数据赋值至实例
      let bardata = {bts_name:[],score:[]}
      var getdata={};
      getBardata().then(response => {
      getdata = response.data.items;
      for (var i = 0; i<  7; i++ ){
        bardata.bts_name[i] = getdata[i]['bts_name'];
        bardata.score[i] = getdata[i]['score'];
      }
      this.chart.setOption({
        xAxis:{
          data:bardata.bts_name,
        },
        series:[{
          data: bardata.score
        }]
      })
      })

    }
  }
}
</script>

简单实现逻辑

  1. Bar没有从父级组件获取数据，通过内部实现数据获取和展示；
  2. 从API组件引入 getBardata 函数，该函数实现从API接口获取数据的功能；
  3. 先定义Echarts的属性，包括图形，主题等；
  4. 利用mount 方法实现等待Echarts的dom渲染
  5. 利用method，定义mount的操包括：初始化Echarts的dom，并将前面定义好的option传入初始化的Echarts实例
  6. 通过API组件的函数获取后台数据，并将Json数据转化为数组
  7. 利用setoption方法将数据传入初始化的Echart实例

估计因为JS的特性，通过API函数获取的数据，离开了函数后就无法实现引用，因此获取数据并赋值的逻辑需要在函数内实现

猜想：可以利用export的方法，将函数获取的数据暴露至组件内的其他方法，实现数据传递
